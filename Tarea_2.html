Para realizar la evaluacin de distintos clasificadores, vamos a crear la funcin run_classifier(), la cual evala un clasificador clf recibido como parmetro, un dataset X,y (features y target) y un nmero de tests llamado num_test. ## run_classifier recibe un clasificador y un dataset (X, y)
## y opcionalmente la cantidad de resultados que se quiere obtener del clasificador

from sklearn.metrics import f1_score, recall_score, precision_score
from sklearn.model_selection import train_test_split
import numpy as np

def run_classifier(clf, X, y, num_tests=100):
    metrics = {'f1-score': [], 'precision': [], 'recall': []}
    
    for _ in range(num_tests):
        X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=.30)
        ### INICIO COMPLETAR AC 
        clf.fit(X_train, y_train) #Entrenamos el modelo con los datos respectivos
        #### TIP: en base a los set de entrenamiento, genere la variable 'predictions' 
        #### que contiene las predicciones del modelo
        predictions = clf.predict(X_test)  #generamos la variable predictions
        ### FIN COMPLETAR AC
        metrics['y_pred'] = predictions
        metrics['f1-score'].append(f1_score(y_test, predictions)) 
        metrics['recall'].append(recall_score(y_test, predictions))
        metrics['precision'].append(precision_score(y_test, predictions))
    
    return metrics

Pregunta 3.2¶

Explique cmo escoge un rbol de decisin el atributo raz.

La idea es que un rbol de decisin sea lo ms pequeo posible para hacer predicciones ms correcta, entonces, para escoger el nodo raz se debe mirar aquella variable o criterio que genere nodos lo ms puros posibles, es decir, que las decisiones a partir de los datos sean lo ms homogneas posible. Uno de los criterios ms conocido es el de information gain, en donde se escoge el nodo que sea ms puro en promedio considerando los distintos subconjuntos de los datos. Se utiliza el clculo de la entropa para cada variable con el fin de cuantificar la informacin, luego se calcula la esperanza para la variable y con este valor se puede calcular el criterio de information gain para decidir cual variable elegir como nodo raz. De esta misma forma iterativa, se pueden decidir cules van siendo los distintos nodos en los distintos niveles.

Pregunta 3.3¶

Explique el problema de optimizacin que resuelve una SVM lineal.

Un problema de Support Vector Machines lineal aborda un problema de clasificacin binario y busca encontrar un hiperplano tal que maximice la distancia entre los puntos de una tupla de un dataset. El problema de optimizacin se define de la siguiente forma:

  • Funcin Objetivo: max $\frac{2}{||w||}$ cambiando w $\Leftrightarrow$ min $\frac{||w||^{2}}{2}$ cambiando w
  • Restriccin: $y_i \cdot (w\cdot x_i + b) \geq 1$, $i=1,2,...,N$

En donde $x_i$ corresponde a un vector que representa a un atributo $i$ de un dataset, la variable $y_i$ corresponde a la decisin binaria de clasificacin, es decir, $y_i \in \{1,-1\}$, siendo cada valor una clasificacin para el dataset. Por otro lado, las variables $w$ y $b$ son las que describen al hiperplano que se encuentra entre los distintos puntos clasificados del dataset.

Pregunta 3.4 (INCOMPLETA)¶

Analizando los resultados obtenidos de cada clasificador, y basndose en las mtricas calculadas. Cul es el mejor clasificador? Qu mtricas observ para tomar esa decisin y por qu? considerando el problema que aborda. Fundamente su respuesta.

Como sabemos precisin y recall se obtienen de la siguiente forma: $$ Precision = \frac{TP}{TP+FP} \\ Recall = \frac{TP}{TP+FN} $$

Por lo que existe un tradeoff entre ambas metricas, pues recall no considera los Falsos Positivos(FP) y precision no considera los Falsos Negativos(FN).

Como estamos frente a un problema de clasificacin sobre cancer de mamas, importan mucho los Falsos Negativos, ya que si una persona lamentablemente tiene cancer y el modelo predice que no tiene, se est cometiendo una equivoacin muy grave, asimismo si se cae en un Falso Negativo tambin, ya que se estara tratando a una persona sana como si tuviera cancer, sin embargo se podra descartar directamente con medicos, por lo que en terminos generales se cree que lo primero es muchsimo peor, al ser una condicin omitida que puede significar la vida de una eprsona.

Por lo que hay que ser muy cauteloso al utilizar estas mtricas e idealmente se espera que el modelo tenga el mximo valor en ambas. Dicho esto se cree que el modelo de Support Vector Machine, al tener el mayor Recall(0.98) se cree que es mejor en este caso, pues es el que comete menor error de Falsos Negativos, la situacin mas grave frente a un diagnostico.

Parte 4: Seleccionar hiperparámetros¶

Los hiperparmetros son parmetros que no se aprenden directamente dentro de los estimadores. En scikit-learn se pasan como argumentos al constructor de las clases, por ejemplo cul kernel usar para Support Vector Classifier, o qu criterio para Decision Tree, etc. Es posible y recomendable buscar en el espacio de hiperparmetros la mejor alternativa.

Tenga en cuenta que es comn que un pequeo subconjunto de esos parmetros pueda tener un gran impacto en el rendimiento predictivo o de clculo del modelo, mientras que otros pueden dejar sus valores predeterminados. Se recomienda leer la documentacin de la clase de estimador para obtener una mejor comprensin de su comportamiento esperado.

GridSearchCV¶

Una alternativa para seleccionar hiperparmetros es GridSearchCV, la cual considera exhaustivamente todas las combinaciones de parmetros. GridSearchCV recibe un estimador, recibe param_grid (un diccionario o una lista de diccionarios con los nombres de los parametros a probar como keys y una lista de los valores a probar), scoring una o varias funciones de puntuacin (score) para evaluar cada combinacin de parametros (opciones vlidas: https://scikit-learn.org/stable/modules/model_evaluation.html) y cv una extrategia para hacer validacin cruzada.

Pregunta 5. 1¶

Por qu aplicar (sub/over) sampling de las clases sobre el conjunto de entrenamiento en lugar de aplicarlo sobre el dataset completo?

En clasificacion se realiza (sub/over) sampling en conjuntos de datos desbalanceados para regular la distribucin de clases de los datos de capacitacin y generar un aprendizaje que impone efectivamente costos de clasificacin errnea no uniformes. Aplicar estas tecnicas sobre el dataset completo provocaria una perdida de generalidad y especificacion del modelo, al testearlo en un dataset perfectamente balanceado, siendo que en la realidad, la gran mayoria de los dataset encontrados estan desbalanceados --> Overfitting.

Pregunta 5.2¶

  • Observe los resultados obtenidos por clase con cada conjunto de entrenamiento, se puede observar alguna diferencia importante?

Podemos notar que la mtrica de recall respecto a la clase 1 es la menor en oversampling, y si recordamos el oversampling se hizo respecto a la clase 1, por lo que esto nos dice que aument la cantidad de Falsos Negativos, es decir, se clasificaron erroneamente la clase 0 que realmente era 1. Asimismo en el subsampling el recall de la clase 1 es mayor al original, lo que puede indicar que disminuy el error por Falsos Negativos, lo que pudo deberse al recorte de la clase 1 para coincidir con la clase 0.

Lo anterior se puede explicar debido a que un clasificador de arbol de decision es muy afectado por el desbalance de clases (esta diseado para optimizar la precision general sin considerar una distribucion relativa de cada clase) llegando a obtener resultados "muy buenos" al no considerar las clases pequeas (Falsos Positivos). Es por esto, que si bien el uso de estas tecnicas de regulacion de muestreo pueden provocar una disminucion en las metricas de desempeo, genera unas mas fieles a la realidad del set de entrenamiento.

  • Indique una desventaja de usar oversampling y una desventaja de usar subsampling en clasificacin.

El oversampling, ademas de aumentar el tiempo de procesado de los datos, puede generar replicas de nuevas instancias de clases a partir de ejemplos ruidosos, de tal manera que la distribucion de la clase minoritaria se modifica en exceso. Esto puede derivar en un incremento desmesurado de las probabilidades de dicha clase provocando que el modelo termine aprendiendo demasiado de lo especifico de los pocos ejemplos (Overfitting). Al aumentar los datos se vuelven mas dispersos, por lo que pierden significado algunas metricas como densidad y distancia (clustering y deteccion de outliers).

En el caso del subsampling, existe el riesgo de eliminar elementos de la muestra potencialmente importantes en el proceso de clasificacion, lo que puede sesgar los resultados y empeorar el anlisis.
